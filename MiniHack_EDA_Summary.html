Sarabun เป็น Open Source Font ที่รองรับทั้งไทย-อังกฤษ และมีน้ำหนักหลายระดับ ดาวน์โหลดได้ฟรีจาก GitHub Google Fonts และต้องรันทุกครั้งที่เปิด Colab ใหม่
Emoji ไม่รองรับ ต้องรันก่อน import plt ใช้ข้อความแทน Emoji ใน annotation

พบ Garbage 3 ประเภท:
• #NAME? — Excel formula error
• (blank) — Pivot Table ว่าง
• - - - xxx — HS Code sub-level prefix
รวม ~16,310 แถว (38.8%)
✓ วิธีแก้
วิเคราะห์สาเหตุก่อนลบ:
• #NAME? → กู้จาก productDetailTH
• (blank) → map HS Code จาก Reference
• - - - → ตัด prefix ออก
แล้วลบ productDetailEN ทิ้ง ใช้ TH เป็นหลัก
★ ทำไมวิธีนี้?
หลักการ "อย่าลบก่อนเข้าใจ" — Garbage แต่ละประเภทมีสาเหตุต่างกัน การลบทันทีทำให้เสียข้อมูลที่ยังกู้คืนได้ การวิเคราะห์สาเหตุก่อนช่วยรักษาข้อมูลได้มากขึ้น
ETL Pipeline ต้องแก้ วิเคราะห์ก่อนลบ กู้คืนได้บางส่วน ใช้ TH เป็น Key หลัก
PROBLEM 03 / Data Consistency
🌍 countryID ของนามิเบียถูกอ่านเป็น NaN
⚠ ปัญหา
รหัส ISO ของนามิเบียคือ "NA" แต่ Pandas อ่าน "NA" เป็น NaN โดยอัตโนมัติ ทำให้รายงาน countryID Missing 25 แถว ทั้งที่ข้อมูลจริงครบ
✓ วิธีแก้
ไม่ต้อง reload ไฟล์ แต่เติมค่ากลับเฉพาะจุด:
df.loc[df['countryNameTH']=='นามิเบีย', 'countryID'] = 'NA'
กรองด้วย countryNameTH ก่อนเพื่อความปลอดภัย
★ ทำไมวิธีนี้?
แก้เฉพาะจุดที่ปัญหา ไม่กระทบข้อมูลอื่น ถ้า reload ไฟล์ใหม่ด้วย keep_default_na=False จะทำให้ NaN จริงๆ ในคอลัมน์อื่นหายไปด้วย ซึ่งอันตรายกว่า
Pandas auto-convert NA→NaN Classic trap! Surgical fix ดีกว่า reload
PROBLEM 04 / Schema Inconsistency
🔢 productDetailEN(204) ≠ productDetailTH(431)
⚠ ปัญหา
สินค้า 1 ชนิด (EN) มีชื่อไทยหลายแบบ เช่น "ปลาทูน่า" / "ทูน่า" / "ปลาทู น่า" ทำให้ Group By ผิดพลาด — สินค้าเดียวกันแตกออกเป็นหลาย Group
✓ วิธีแก้
เลือก ชื่อ TH ที่ใช้บ่อยที่สุด (mode) ของแต่ละ EN มาเป็น Canonical Name แล้ว Map กลับเข้า DataFrame ก่อนลบ EN ทิ้ง เพื่อลด unique TH จาก 431 → ต่ำกว่า
★ ทำไมวิธีนี้?
ใช้ Majority Vote (mode) เพราะชื่อที่ใช้บ่อยที่สุด มักเป็นชื่อมาตรฐานที่เจ้าของข้อมูลตั้งใจใช้ และหลีกเลี่ยง Bias จากการเลือกชื่อเอง
Dirty Master Data Mode-based Standardize ใช้ EN เป็น Join Key TH สำหรับ Display
PROBLEM 05 / Column Management
📝 tradeflow เป็นตัวเลข (1/2) อ่านยาก
⚠ ปัญหา
tradeflow = 1 หรือ 2 ไม่บอก Context ต้องจำตลอดว่า 1=Import หรือ 2=Export และ Data Dictionary ระบุสลับกับที่คาดไว้ (1=Import ไม่ใช่ Export)
✓ วิธีแก้
Map ตัวเลขเป็นข้อความ:
df['tradeflow'].map({1:'Import', 2:'Export'})
แล้วแทนที่ค่าเดิมใน Column เดิม ไม่ต้องสร้าง Column ใหม่
★ ทำไมวิธีนี้?
Self-documenting Code — Code ที่อ่านแล้วเข้าใจได้เอง ลด Human Error จากการจำตัวเลข และทำให้ Visualization Label ถูกต้องอัตโนมัติโดยไม่ต้องแปลงซ้ำ
อ่านไม่ออก Magic Number Self-documenting Label ถูกอัตโนมัติ
PROBLEM 06 / Feature Engineering
📅 ไม่มี Column Quarter และ Season สำหรับ Seasonal Analysis
⚠ ปัญหา
ข้อมูลมีแค่ month (1-12) แต่ต้องการวิเคราะห์ ตาม Quarter (Q1-Q4) และ ฤดูกาลไทย (ร้อน/ฝน/หนาว) ซึ่งสะท้อน Pattern การค้าได้ดีกว่าการดูรายเดือนแบบ Raw
✓ วิธีแก้
สร้าง 2 Column ใหม่ ด้วย .map():
• quarter: Q1-Q4 (3 เดือน/ไตรมาส)
• season: ร้อน(มี.ค-พ.ค) / ฝน(มิ.ย-ต.ค) / หนาว(พ.ย-ก.พ)
★ ทำไมวิธีนี้?
Feature Engineering — สร้าง Feature ใหม่จากข้อมูลที่มี เพื่อเพิ่ม Granularity ในการวิเคราะห์ ฤดูกาลไทยสัมพันธ์กับ ฤดูจับสัตว์น้ำ โดยตรง
Feature Engineering Domain Knowledge ฤดูกาลไทย ≠ สากล
PROBLEM 07 / Missing Reference Data
📦 (blank) ใน productDetailTH ไม่รู้ว่าสินค้าอะไร
⚠ ปัญหา
พบ 34 แถว ที่ productDetailTH = (blank) ไม่รู้ชื่อสินค้า ทำให้วิเคราะห์แบบ Product-based ไม่ได้ แต่ยังมี heading11 (HS Code) เป็นตัวระบุ
✓ วิธีแก้
ค้น HS Code จาก แหล่งอ้างอิงภายนอก (กรมศุลกากร) แล้ว Manual Fill:
HS 3032900000 → "ปลาแซลมอนชนิดอื่น แช่แข็ง ไม่รวมตับและไข่ปลา"
★ ทำไมวิธีนี้?
HS Code เป็น มาตรฐานสากล (World Customs Organization) ใช้เป็น Foreign Key ในการ Lookup ชื่อสินค้าจาก Reference Table วิธีนี้ถูกต้องกว่าการเดาหรือทิ้งข้อมูล
Missing Master Data HS Code = Foreign Key Manual + Verified Source

PART 03 Framework การตรวจสอบ Data Quality
มิติ คำถามที่ต้องถาม เครื่องมือที่ใช้ พบใน MiniHack 1
Completeness ข้อมูลครบทุกแถว/คอลัมน์ไหม? .isnull().sum() countryID missing 25 แถว (นามิเบีย)
Validity ค่าอยู่ในช่วงที่ถูกต้องไหม? price ≤ 0 check price = 0 พบบางแถว
Consistency ข้อมูลสอดคล้องกันข้ามคอลัมน์? .nunique() cross-check EN(204) ≠ TH(431), countryID(196) ≠ TH(197)
Accuracy ข้อมูลตรงกับความเป็นจริงไหม? HS Code Reference Lookup (blank) ใน TH → map จากกรมศุลกากร
Uniqueness มีข้อมูลซ้ำซ้อนไหม? .duplicated() ชื่อ TH หลายแบบสำหรับสินค้าเดียว
Timeliness ข้อมูลทันสมัยพอไหม? ETL_DATE check ETL_DATE มีค่าเดียว (12/3/2026 05:59)

PART 05 Lessons Learned — บทเรียนสำคัญ
🧭
Data Cleaning ≠ Data Deleting
Cleaning คือการทำให้ข้อมูลน่าเชื่อถือ ไม่ใช่แค่ลบแถวที่ "ดูผิดปกติ" การเข้าใจที่มาของข้อมูลช่วยรักษาข้อมูลได้มากกว่า
🔬
EDA = Journalism
ตั้งคำถาม → หาหลักฐาน → ตรวจสอบสมมติฐาน → สรุปผล วิธีคิดเหมือนนักข่าวสืบสวน ไม่ใช่แค่รัน Code
⚡
Domain Knowledge สำคัญเท่า Code
รู้ว่า ต.ค. คือฤดูทะเลสงบในอ่าวไทย ทำให้ตีความ Peak ได้ถูกต้อง Code อย่างเดียวบอกแค่ตัวเลข
🎯
ตัวเลข 38.8% บอกอะไร?
Garbage 38% ไม่ใช่แค่ปัญหาข้อมูล แต่บ่งชี้ปัญหาใน ETL Pipeline ที่ต้นทาง การแก้ต้นเหตุดีกว่าแก้ปลายเหตุทุกครั้ง
📐
Iterative ดีกว่า Linear
พบ (blank) ใน Section 2 → กลับไปแก้ใน Section 3 → ตรวจสอบอีกครั้ง กระบวนการวนซ้ำให้ผลดีกว่าทำครั้งเดียว
📣
Insight ไม่มีค่าถ้าไม่นำไปใช้
ทุก Insight ต้องตอบ "So what?" และ "Now what?" ถึงจะมีคุณค่าต่อองค์กร Data สำหรับ Data เฉยๆ ไม่ใช่ Data Science
